# 🔐 Логика пароля при регистрации через Telegram

## 📋 Проблема

При регистрации через Telegram автоматически генерировался случайный UUID пароль, который пользователь не знал. Это создавало проблемы:
- Пользователь не мог войти через обычную форму
- Нет возможности восстановить доступ, если Telegram недоступен

## ✅ Реализованное решение

### 1. Генерация читаемого пароля

Вместо UUID теперь генерируется **читаемый пароль из 8 символов**:
- Буквы (A-Z, a-z) без похожих символов (I, l, O, 0)
- Цифры (2-9)
- Пример: `Kp3mN8xQ`

### 2. Модальное окно после регистрации

После успешной регистрации через Telegram показывается модальное окно с:

**Информация:**
- ✅ Временный пароль (крупным шрифтом)
- ✅ Кнопка "Копировать" для удобства
- ✅ Предупреждение сохранить пароль

**Возможности:**
- 🔑 **Установить свой пароль** - форма с двумя полями
- ⏭️ **Пропустить** - можно установить позже в профиле

### 3. Отправка пароля на email

Если у пользователя есть реальный email (не placeholder), пароль отправляется на почту автоматически.

### 4. Установка пароля в профиле

Пользователь может установить пароль позже в разделе "Безопасность" профиля.

## 🔄 Поток регистрации

1. Пользователь нажимает "Войти через Telegram"
2. Авторизуется в Telegram
3. Система создает аккаунт с временным паролем
4. Показывается модальное окно с паролем
5. Пользователь может:
   - Скопировать пароль
   - Установить свой пароль
   - Пропустить (установить позже)

## 📝 Технические детали

### Генерация пароля

```java
private String generateReadablePassword() {
    String chars = "ABCDEFGHJKLMNPQRSTUVWXYZabcdefghijkmnpqrstuvwxyz23456789";
    // Генерирует 8 символов
}
```

### Endpoint для установки пароля

```
POST /auth/set-password
Body: { "password": "новый_пароль" }
```

### Отправка на email

Метод `sendTelegramPasswordEmail()` отправляет пароль только если:
- Email не пустой
- Email не является placeholder (`telegram_*@ruomi.fi`)

## 🎯 Рекомендации для пользователей

1. **Сохраните пароль** в безопасном месте
2. **Установите свой пароль** для удобства
3. **Используйте Telegram** для быстрого входа
4. **Пароль нужен** для входа через обычную форму

## 🔒 Безопасность

- Пароль показывается только один раз (в модальном окне)
- Пароль отправляется на email только если это реальный email
- Пользователь может установить свой пароль в любой момент
- Пароль хранится в захешированном виде в БД

## ✅ Проверка

После регистрации через Telegram:
1. Должно появиться модальное окно с паролем
2. Пароль должен быть читаемым (8 символов)
3. Можно скопировать пароль
4. Можно установить свой пароль
5. Можно пропустить и установить позже
