# Система автоматического обновления

## Описание

Система автоматического обновления позволяет обновлять приложение прямо из админ-панели. Обновления загружаются с GitHub Releases в виде готовых JAR файлов.

## Возможности

- ✅ Автоматическая проверка новых версий каждый час
- ✅ Уведомление о новой версии в админ-панели
- ✅ Обновление в один клик из веб-интерфейса
- ✅ Автоматическое создание резервных копий
- ✅ Откат при ошибке обновления
- ✅ Прогресс-бар и логи процесса обновления

## Настройка

### 1. Настройка GitHub репозитория

1. Создайте репозиторий на GitHub (если еще не создан)
2. Добавьте настройки в `application.properties`:

```properties
# Application Update Configuration
app.version=0.0.1-SNAPSHOT
app.update.github.owner=ваш-username-на-github
app.update.github.repo=название-репозитория
app.update.github.token=ваш-github-token  # Опционально, для приватных репозиториев
app.update.download.path=./updates
app.update.backup.path=./backups
app.jar.path=./target/doska-0.0.1-SNAPSHOT.jar
```

### 2. Создание релиза на GitHub

#### Автоматический способ (рекомендуется)

Используйте готовые скрипты для автоматической загрузки:

**Windows:**
```bash
release.bat
```

**Linux/Mac:**
```bash
chmod +x release.sh
./release.sh
```

Скрипт автоматически:
- ✅ Соберет проект
- ✅ Создаст релиз на GitHub
- ✅ Загрузит JAR файл

**Требования:**
- GitHub CLI (`gh`) - установите с [cli.github.com](https://cli.github.com)
- Или GitHub Personal Access Token (скрипт запросит его)

Подробнее: см. [RELEASE_GUIDE.md](RELEASE_GUIDE.md)

#### Ручной способ

1. Соберите JAR файл:
```bash
mvn clean package -DskipTests
```

2. Создайте релиз на GitHub:
   - Перейдите в раздел "Releases" вашего репозитория
   - Нажмите "Create a new release"
   - Укажите версию (например, `v1.0.0` или `1.0.0`)
   - Добавьте описание релиза
   - Загрузите JAR файл из `target/doska-0.0.1-SNAPSHOT.jar`
   - Опубликуйте релиз

**Важно:** JAR файл должен иметь расширение `.jar` и не содержать в названии `sources` или `javadoc`.

### 3. Настройка пути к JAR файлу

В зависимости от того, как вы запускаете приложение, нужно настроить `app.jar.path`:

- **Для разработки (mvn spring-boot:run):** `./target/doska-0.0.1-SNAPSHOT.jar`
- **Для продакшена (java -jar):** путь к вашему JAR файлу, например `./app.jar` или `/opt/doska/app.jar`

### 4. Создание директорий

Система автоматически создаст директории `./updates` и `./backups`, но вы можете создать их вручную:

```bash
mkdir updates backups
```

## Использование

### Доступ к странице обновлений

1. Войдите в систему как администратор
2. Перейдите в меню пользователя → "Обновления"
3. Или напрямую: `http://localhost:8080/admin/updates`

### Проверка обновлений

- **Автоматически:** Система проверяет обновления каждый час
- **Вручную:** Нажмите кнопку "Проверить обновления" на странице

### Обновление приложения

1. Если доступна новая версия, появится кнопка "Обновить"
2. Нажмите кнопку "Обновить"
3. Подтвердите действие
4. Следите за прогрессом обновления
5. После успешного обновления приложение нужно будет перезапустить вручную

**Важно:** После обновления JAR файла приложение нужно перезапустить вручную. Система не может автоматически перезапустить сама себя.

## Процесс обновления

1. **Проверка** - Проверяется наличие новой версии
2. **Резервное копирование** - Создается backup текущего JAR файла
3. **Скачивание** - Скачивается новая версия с GitHub
4. **Установка** - Новый JAR заменяет текущий
5. **Обновление БД** - Обновляется информация о версии в базе данных
6. **Откат** - При ошибке автоматически восстанавливается предыдущая версия

## Безопасность

- Доступ к обновлениям имеют только пользователи с ролями `ADMIN` и `SUPER_ADMIN`
- Резервные копии сохраняются перед каждым обновлением
- При ошибке автоматически выполняется откат
- GitHub токен хранится в `application.properties` (не коммитьте его в Git!)

## Устранение неполадок

### Ошибка "Не удалось получить информацию о последней версии"

- Проверьте настройки `app.update.github.owner` и `app.update.github.repo`
- Убедитесь, что репозиторий публичный или используйте GitHub токен
- Проверьте, что на GitHub есть хотя бы один релиз

### Ошибка "URL для скачивания не найден в релизе"

- Убедитесь, что в релизе загружен JAR файл
- JAR файл должен иметь расширение `.jar`
- Имя файла не должно содержать `sources` или `javadoc`

### Ошибка "Текущий JAR файл не найден"

- Проверьте настройку `app.jar.path` в `application.properties`
- Убедитесь, что путь указан правильно относительно рабочей директории приложения

### Обновление завершилось, но изменения не применились

- Приложение нужно перезапустить вручную после обновления
- Проверьте, что новый JAR файл действительно заменил старый
- Проверьте логи приложения

## Структура файлов

```
doska/
├── updates/          # Скачанные обновления
├── backups/          # Резервные копии
└── target/           # Собранные JAR файлы (для разработки)
    └── doska-0.0.1-SNAPSHOT.jar
```

## API эндпоинты

- `GET /admin/updates` - Страница управления обновлениями
- `GET /admin/updates/api/check` - Проверка обновлений
- `GET /admin/updates/api/status` - Статус текущего обновления
- `POST /admin/updates/api/update` - Запуск обновления
- `GET /admin/updates/api/releases` - Список всех релизов

## Примечания

- Система не может автоматически перезапустить приложение после обновления
- Рекомендуется делать резервные копии базы данных перед обновлением
- Тестируйте обновления на тестовом сервере перед применением на продакшене
