# 🔑 Настройка GitHub Token для обновлений

## Проблема: Ошибка 403 Forbidden

Если вы видите ошибку **403 Forbidden** при попытке проверить обновления, это означает, что приложение не может получить доступ к GitHub API.

## Причины ошибки 403:

1. **Репозиторий приватный** - для приватных репозиториев требуется токен
2. **Токен не настроен** - в `application.properties` не указан `app.update.github.token`
3. **Токен неверный** - токен истек или неверный
4. **Токен не имеет прав** - токен не имеет прав на чтение репозитория

## Решение: Создание GitHub Token

### Шаг 1: Создайте Personal Access Token

1. Перейдите на GitHub: https://github.com/settings/tokens
2. Нажмите **"Generate new token"** → **"Generate new token (classic)"**
3. Заполните форму:
   - **Note**: `ruomi-updates` (любое название)
   - **Expiration**: Выберите срок действия (рекомендуется "No expiration" для продакшена)
   - **Scopes**: Отметьте:
     - ✅ `repo` (для приватных репозиториев)
     - ✅ `public_repo` (для публичных репозиториев, если репозиторий публичный)
4. Нажмите **"Generate token"**
5. **Скопируйте токен** (он показывается только один раз!)

### Шаг 2: Настройте токен в application.properties

Отредактируйте файл `application.properties`:

```properties
# Application Update Configuration
app.update.github.owner=mifedweb
app.update.github.repo=ruomi
app.update.github.token=ваш_токен_здесь
```

**Важно:** Замените `ваш_токен_здесь` на реальный токен, который вы скопировали.

### Шаг 3: Перезапустите приложение

```bash
sudo systemctl restart ruomi
```

## Для публичных репозиториев

Если репозиторий **публичный**, токен не обязателен, но рекомендуется для:
- Увеличения лимита запросов (60 запросов/час без токена, 5000 с токеном)
- Более стабильной работы

## Проверка настроек

После настройки токена:

1. Перейдите в админ-панель → **Обновления**
2. Нажмите **"Проверить обновления"**
3. Если все настроено правильно, вы увидите информацию о последней версии

## Безопасность

⚠️ **Важно:** 
- Никогда не коммитьте токен в Git!
- Храните токен в секрете
- Используйте переменные окружения для продакшена (см. ниже)

## Использование переменных окружения (рекомендуется)

Вместо хранения токена в `application.properties`, используйте переменную окружения:

```bash
# В systemd service файле или .env
export APP_UPDATE_GITHUB_TOKEN=ваш_токен_здесь
```

Или в `application.properties`:
```properties
app.update.github.token=${APP_UPDATE_GITHUB_TOKEN:}
```

## Устранение неполадок

### Ошибка все еще появляется?

1. **Проверьте токен:**
   ```bash
   # На сервере
   grep "app.update.github.token" /opt/ruomi/application.properties
   ```

2. **Проверьте права токена:**
   - Убедитесь, что токен имеет права `repo` или `public_repo`
   - Проверьте, что токен не истек

3. **Проверьте репозиторий:**
   - Убедитесь, что репозиторий существует: `https://github.com/mifedweb/ruomi`
   - Проверьте, является ли он приватным

4. **Проверьте логи:**
   ```bash
   sudo journalctl -u ruomi -f | grep -i "github\|403\|forbidden"
   ```

5. **Протестируйте токен вручную:**
   ```bash
   curl -H "Authorization: token ваш_токен" \
        https://api.github.com/repos/mifedweb/ruomi/releases/latest
   ```

## Альтернатива: Сделать репозиторий публичным

Если вы не хотите использовать токен, можно сделать репозиторий публичным:

1. Перейдите в настройки репозитория на GitHub
2. Прокрутите вниз до "Danger Zone"
3. Нажмите "Change visibility" → "Make public"

После этого токен не потребуется, но лимит запросов будет 60/час.
