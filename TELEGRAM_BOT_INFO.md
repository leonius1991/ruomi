# 🤖 Информация о Telegram боте уведомлений

## 📋 Статус бота

Telegram бот для уведомлений автоматически регистрируется при запуске приложения, если:
1. В `application.properties` указан токен бота: `telegram.bot.token`
2. Токен не равен `YOUR_BOT_TOKEN_HERE` или пустой строке

## 🔧 Конфигурация

В `application.properties` должны быть указаны:

```properties
# Telegram Bot Configuration
telegram.bot.username=ruomi_fi_bot
telegram.bot.token=8221655282:AAG6AFmgpXcfPZQedsYIBbcbEHWJNAlWtig
```

## ✅ Проверка работы бота

### 1. Проверка логов при запуске

При запуске приложения в логах должно быть:
- `✅ Telegram bot registered successfully!` - бот успешно зарегистрирован
- `⚠️ Telegram bot not configured. Skipping registration.` - бот не настроен
- `❌ Failed to register Telegram bot: ...` - ошибка регистрации

### 2. Проверка через Telegram

1. Найдите бота в Telegram: `@ruomi_fi_bot`
2. Отправьте команду `/start`
3. Бот должен ответить приветственным сообщением

### 3. Проверка через логи приложения

```bash
# На сервере
sudo journalctl -u ruomi -f | grep -i telegram
```

## 🎯 Функции бота

Бот используется для:
1. **Уведомления о новых сообщениях** - когда пользователь получает личное сообщение
2. **Уведомления о подписках** - когда появляются новые объявления в подписанных категориях
3. **Рассылки** - администраторы могут отправлять массовые сообщения
4. **Взаимодействие с пользователями** - команды, меню, ответы

## 🔍 Диагностика проблем

### Проблема: Бот не отвечает

**Решение:**
1. Проверьте токен в `application.properties`
2. Проверьте логи приложения на наличие ошибок
3. Убедитесь, что бот запущен в BotFather
4. Проверьте, что приложение имеет доступ к интернету

### Проблема: "Failed to register Telegram bot"

**Возможные причины:**
- Неверный токен
- Бот уже зарегистрирован в другом приложении
- Проблемы с сетью

**Решение:**
1. Проверьте токен в BotFather
2. Убедитесь, что бот не используется в другом месте
3. Перезапустите приложение

### Проблема: Бот не отправляет уведомления

**Решение:**
1. Убедитесь, что у пользователя есть `telegramId` в базе данных
2. Проверьте логи на наличие ошибок отправки
3. Убедитесь, что пользователь начал диалог с ботом (`/start`)

## 📝 Команды бота

- `/start` - Начать работу с ботом
- `/help` - Справка
- `/menu` - Главное меню
- `/subscriptions` - Управление подписками
- `/messages` - Личные сообщения

## 🔄 Обновление конфигурации

После изменения токена в `application.properties`:
1. Перезапустите приложение: `sudo systemctl restart ruomi`
2. Проверьте логи на успешную регистрацию

## 📞 Поддержка

Если бот не работает:
1. Проверьте логи: `sudo journalctl -u ruomi -n 100`
2. Проверьте конфигурацию: `grep telegram /opt/ruomi/application.properties`
3. Проверьте статус бота в BotFather
