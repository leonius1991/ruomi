# 🔄 PTR запись (обратная DNS) - подробное объяснение

## ❓ Что такое PTR запись?

PTR (Pointer) запись - это обратная DNS запись. Она связывает IP адрес с доменным именем.

**Обычная DNS:** домен → IP (A запись)
**Обратная DNS:** IP → домен (PTR запись)

## 🎯 Зачем нужна PTR запись?

**Без PTR записи:**
- ❌ Многие почтовые серверы будут отклонять ваши письма
- ❌ Письма будут попадать в спам
- ❌ Gmail, Outlook и другие будут блокировать письма

**С PTR записью:**
- ✅ Письма доставляются надежнее
- ✅ Меньше вероятность попадания в спам
- ✅ Профессиональный вид

## 🔍 У какого провайдера настраивать?

### Ответ: У вашего хостинг-провайдера (VPS провайдера)

PTR запись настраивается **НЕ у DNS провайдера** (где вы настраиваете домен), а у **провайдера, который предоставил вам сервер/VPS**.

### Как узнать, кто ваш провайдер?

```bash
# Узнайте ваш IP адрес
hostname -I
# или
ip addr show

# Проверьте, кто владеет IP
whois YOUR_SERVER_IP
```

### Примеры провайдеров:

- **DigitalOcean** - настраивается в панели управления
- **Linode** - настраивается в панели управления
- **Vultr** - настраивается в панели управления
- **Hetzner** - настраивается в панели управления
- **AWS** - через Route 53 или поддержку
- **Google Cloud** - через консоль
- **Azure** - через портал
- **Локальный хостинг** - через техподдержку

## 📝 Как настроить PTR запись

### Вариант 1: Через панель управления провайдера

Большинство VPS провайдеров позволяют настроить PTR в панели:

1. Войдите в панель управления вашего провайдера
2. Найдите раздел "Reverse DNS" или "PTR Records"
3. Добавьте запись:
   - **IP адрес:** `YOUR_SERVER_IP`
   - **Hostname:** `mail.ruomi.fi`

### Вариант 2: Через техподдержку

Если в панели нет такой опции:

1. Свяжитесь с техподдержкой вашего провайдера
2. Попросите настроить обратную DNS (PTR запись)
3. Укажите:
   - IP адрес вашего сервера
   - Доменное имя: `mail.ruomi.fi`

**Пример запроса:**
```
Здравствуйте!

Прошу настроить обратную DNS (PTR запись) для моего сервера:
IP адрес: 123.45.67.89
Hostname: mail.ruomi.fi

Спасибо!
```

## ✅ Проверка PTR записи

После настройки проверьте:

```bash
# Проверка обратной DNS
dig -x YOUR_SERVER_IP

# Или
host YOUR_SERVER_IP

# Должно вернуть: mail.ruomi.fi
```

**Онлайн проверка:**
- https://mxtoolbox.com/ReverseLookup.aspx
- Введите ваш IP адрес

## ⚠️ Важно!

1. **PTR запись настраивается у провайдера сервера**, а не у DNS провайдера домена
2. **Должна совпадать с A записью** - если PTR указывает на `mail.ruomi.fi`, то A запись `mail.ruomi.fi` должна указывать на этот IP
3. **Может занять время** - обычно настраивается в течение 24-48 часов
4. **Некоторые провайдеры не предоставляют PTR** - в этом случае письма могут попадать в спам

## 🔍 Как узнать, кто ваш провайдер?

### Шаг 1: Узнайте IP адрес

```bash
hostname -I
```

### Шаг 2: Проверьте whois

```bash
whois YOUR_SERVER_IP
```

Или онлайн: https://whois.net/

### Шаг 3: Найдите провайдера

В выводе whois найдите:
- `OrgName:` - название организации
- `NetName:` - название сети
- `descr:` - описание

Это и есть ваш провайдер.

## 📞 Примеры для популярных провайдеров

### DigitalOcean

1. Войдите в панель: https://cloud.digitalocean.com/
2. Перейдите в **Networking** → **Domains**
3. Или используйте API: https://docs.digitalocean.com/reference/api/api-reference/#tag/Domains

### Hetzner

1. Войдите в панель: https://console.hetzner.cloud/
2. Перейдите в **Networking** → ваш сервер
3. Найдите раздел **Reverse DNS**

### Vultr

1. Войдите в панель: https://my.vultr.com/
2. Перейдите в **Products** → ваш сервер
3. Найдите раздел **Reverse DNS**

### AWS

1. Через Route 53 (если используете)
2. Или через техподдержку AWS

## 🆘 Если провайдер не предоставляет PTR

Если ваш провайдер не может настроить PTR запись:

1. **Используйте готовые почтовые сервисы** (Yandex.Connect, SendGrid) - они уже имеют правильные PTR
2. **Используйте релей** - отправляйте через другой SMTP сервер
3. **Смените провайдера** - выберите того, кто предоставляет PTR

## 📝 Пример полной настройки

```
1. A запись: mail.ruomi.fi → 123.45.67.89
2. PTR запись: 123.45.67.89 → mail.ruomi.fi
3. MX запись: ruomi.fi → mail.ruomi.fi
4. SPF запись: ruomi.fi → "v=spf1 mx a:mail.ruomi.fi ~all"
5. DKIM запись: default._domainkey.ruomi.fi → "v=DKIM1; ..."
```

Все записи должны быть согласованы!
